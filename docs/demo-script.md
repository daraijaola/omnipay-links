# Loom demo script

1. Open OmniPay Links and explain the one-liner: any-token payment links for Telegram merchants, settled through STON.fi Omniston.
2. Show the landing page and the judge demo checkout.
3. Open `/create`, enter Alice Studio as merchant, choose 50 USDT settlement, and generate a link.
4. Click share/copy and explain this link can be sent in Telegram DMs, groups, channels, and mini-app flows.
5. Open the checkout as Bob, select TON/STON/NOT as the payment asset, and fetch an Omniston quote.
6. Point out the source asset, settlement asset, expected output, route, fee, slippage, ETA, and quote status.
7. Confirm the payment. If running without wallet access, call out demo fallback mode clearly; if running with TON Connect, show the wallet confirmation.
8. Show the receipt and Telegram share receipt action.
9. Open the merchant dashboard and show invoice metrics and recent payment state.
10. Close with the pitch: this is not a swap UI; it is Telegram commerce with Omniston as liquidity/routing infrastructure.
